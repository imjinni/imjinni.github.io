## Jekyll

`Jekyll`은 여러 텍스트 파일로부터 정적 웹사이트 구축을 위한 파일을 생성해주는 프레임워크이다.

쉽게 말해서 `Jekyll`은 다양한 형식의 텍스트 파일을 웹 페이지 구성 요소인 `HTML`, `CSS`로 변환해준다. 사실 나도 필요할때 필요한 기능만 사용하는터라 잘 알진 못한다.<br/>
특히 내가 좋아하는 기능은 `Markdown`으로 작성한 문서를 `HTML` 파일로 변환시켜주는 기능이다.<br/>
물론 단순히 변환만으로 끝나는 것이 아니다. 정적 웹사이트를 만든다는 목적에 맞게, 변환된 내용이 그럴듯한 웹 화면으로 보이게끔 자동화된 방법으로 뷰를 꾸미는 것을 돕는다.<br/>
또한 인터넷에 아주 많은 `Jekyll Theme`이 올라와 있으므로 웹 페이지를 빈 화면으로부터 하나하나 다 만들 필요가 없으며, 적당한 테마를 다운받고 가져다 쓰면 된다. 이쁜 테마를 선택했고 뷰가 맘에 든다면 별다른 수정없이 그저 포스팅만 해도 된다.

예를들어, 내 블로그의 데스크탑 모드 화면에서 상단 메뉴, 왼쪽 사이드바는 내가 직접 `HTML`, `CSS`를 수정 및 적용했고 `Jekyll`의 도움으로 다른 뷰에서도 항상 이 2가지가 화면에 보이도록 자동화했다. (반응형 웹으로 만들었기 때문에 모바일 모드로 보면 좀 다르다.)<br/>
동시에 포스트의 글 부분은 `Markdown`으로 작성하였으며 이것을 `Jekyll`이 `HTML` 파일로 변환했고, 나는 그저 `Markdown` 포스트가 `HTML`로 변환된 이후를 가정하고 항상 화면의 중앙에 보이게끔 배치하였다.

## 블로그 만들기 시작

### 1. 블로그 생성

블로그는 HTML, CSS, Javascript를 구성 요소로 해서 만들어진다고 보면된다. 일단 구성 요소들을 만들었다면 브라우저가 구성 요소들을 인식하고 우리가 아는 일반적인 웹 페이지처럼 시각적으로 보여주게된다. 여기까지는 __내__ 컴퓨터에서 __나만__ 볼수 있는 블로그이다.<br/>하지만 우리의 목적은 블로그를 다른 사람들과 공유하는 것이므로, 인터넷이 필요하다.

### 2. 개발 환경 구성

블로그를 구축하기 위해 필요한 환경은 아래와 같다. 환경을 갖춰 놓고 가자.

* Ruby

    객체 지향 프로그래밍 언어. 나는 윈도우 OS로 개발하므로 Ruby Installer를 공식 사이트의 [다운로드 페이지](https://rubyinstaller.org/downloads/)를 통해 다운로드 받았다.<br/>다운로드 목록의 Installer들은 `Devkit`을 포함된 목록과 그렇지 않은 목록이 있는데, 나는 포함된 목록 중 옆에 _stable하다, recommend 한다_ 같은 언급이 있는 버전을 다운로드 했다.<br/>
    또 설치 중 이어서 아래 그림처럼 toolchain 등을 다운로드 할거냐 묻는다.

    ![Ruby Installer CMD](/assets/images/ETC/Blog/RubyInstaller.png)

    이건 테스트 해봤다. 각 항목의 용도는 사실 모르겠고, 찾아봐야하는데 좀 귀찮다... 아무튼 1번 Base Installation이면 되더라.

* Jekyll, Bundler

    위에서 설명했던 프레임워크와 그 부속 도구이다. 이들은 Ruby로 기반 프로그램이므로 Ruby를 설치한 이후 빠르게 설치 가능하다. Ruby 언어에도 Java의 maven, Python의 pip, Unix 계열의 apt-get, yum과 같은 패키지 자동 관리 도구가 있다. 바로 `gem` 이다. 아래와 같은 명령어를 명령어 프롬프트에 입력하여 Jekyll과 Bundler를 자동 Install 하자. (나는 윈도우용 Git bash를 이용하고 있다.)

    ```shell
    gem install jekyll bundler
    ```

* VSCode

    마지막으로 앞으로 모든 개발, 테스트를 진행할 IDE를 설치하자. IDE가 특정 언어에 국한하지 않고, HTML, CSS, SASS, Markdown 등을 다룰 수 있어서 편리하다. 각자 다른 편리한 IDE가 있다면 그걸 써도 좋다. 나는 일하면서 Pycharm에서 VSCode로 갈아타려는 시도를 꽤 해봤으므로 VSCode가 제법 친숙하다.

    ![IDE VSCode](/assets/images/ETC/Blog/IDE_VSCode.png)

    위 VSCode 이미지에서 보이는것만 봐도 지원하는 기능이 이런것들이 있다.

    * 다양한 파일 형태에 대응하는 개발 환경<br/>
      VSCode의 사용자들이 만든 패키지들이 엄청 많다!! 개인적이고 효과적인 개발 환경을 꾸며서 사용할 수 있다.

    * 명령 프롬프트 연동<br/>나는 Git bash를 대신 붙여서 사용한다. 따라서 개발하다가 바로 jekyll 명령어나, git 명령어를 사용할수 있다. 그래서 나는 두개의 명령 프롬프트를 켜고, 하나는 테스트 서버를 구동, 하나는 git 작업을 한다. 유용유용! >_<

    * 약간의 Git Support<br/>
      VSCode에서 Git 작업을 GUI로 약간 보조 받을 수 있다. 나는 그냥 변경된 파일이 뭐뭐 있는지 보는 용도(왼쪽 파일 목록에서 색깔이 변함)로 사용한다.

### 4. 내 Git Page 생성하기

이제 내 블로그를 등록해보자. 아주 간단하다.
아 참고로 Git의 레파지토리는 1GB?까지 용량을 업로드하기까지 공짜로 이용가능하다. 즉, 내가 만든 블로그도 레파지토리의 하나인데 이또한 최대 1GB까지의 파일, 글, 이미지 등의 업로드가 가능하다는 의미이다. 그 이상이 되면.. 결제를 하자! 얼마 안했던걸로 안다. <span class='md-monologue'>얼른 블로그를 키우고 구글 AD Sense를 넣어서 무한 동력 블로그가 되자.</span>

자 먼저 Github에 로그인 한다.

우리가 만들 블로그는 Github의 레파지토리 중 하나로 만들어진다. 여기서 중요한 점은 레파지토리의 이름이 꼭 `[내 Github Username].github.io` 이어야 한다! Github 계정 ID가 아니고 계정 내에서 사용하는 유저명이다! 예를들어, 내 계정 ID는 gmail의 계정이지만 나의 유저명은 `seungwubaek` 이다. 따라서 내 Git Page의 레파지토리 이름은 `seungwubaek.github.io` 이다.

다 만들고서 내 블로그에 접속하려면 `https://seungwubaek.github.io`로 접속한다.

여기서 __선택1__ ! 완전 빈 깡통의 블로그를 만들겠다면, 아래처럼 왼쪽 상단의 New 버튼을 클릭해서 새로운 레파지토리를 생성하는 화면으로 가자.

![](/assets/images/ETC/Blog/github_page_new_repository.png){:style="max-height: 15em;"}

__선택2__ ! 그러나 빈 깡통이 아닌 구성품이 갖춰져있는 `Jekyll Theme`을 이용하겠다면 몇가지 방법이 있다.<br/>위 선택으로 만든 빈 깡통 레파지토리에 어디선가 `Theme`을 다운로드 받아 내 레파지토리로 업로드 하거나 누군가 만들어 놓은 Git Repository를 Fork 해서 가져다 쓰는 방법이 대표적이다. `Theme`은 검색하면 아주 많이 나온다! 각자 맘에 드는걸 찾자.

나는 이 단계에서 두번째 방법을 선택했다. 그리고 내 `Jekyll Theme` 픽은 [minimal-mistakes](https://github.com/mmistakes/minimal-mistakes)이다. 따라서 이 `Theme`을 쓴다는 가정하에 진행하도록 하겠다. 이 테마를 픽한 이유는 다음 섹션을 보면 알 수 있다.

근데 잘 모르는 채로 처음부터 __선택1__ 을 선택하는 것은 정말 추천하지 않는다. 사실 좀 반대한다.. __선택2__ 로 가자..

### 5. Jekyll Theme 가져오기

2020년 10월 기준 가장 Fork가 많았던 [mmistakes theme](https://github.com/mmistakes/minimal-mistakes)을 가져왔다. 이 테마는 운영자가 오래전부터 지금까지 꾸준히 활동하면서 개선 하고 있는 테마이다. 그래서 최근의 트렌드가 가장 잘 반영되었을 것이라 생각했다. 당장 `Google Analytics` 연동, `댓글` 기능이 구현 돼 있는것 만으로도 만족한다.<br/>
그리고 이 테마를 사용한다면, HTML, CSS 등 화면 구성은 하나도 건드리지 않고 설정 파일만 좀 작성하면 블로그 포스팅 작업을 바로 시작할 수 있다.

자 이제 Fork를 하자. mmistakes repository에 들어가서 오른쪽 상단에 Fork 버튼이 있다. 클릭하자.

![](/assets/images/ETC/Blog/fork_1.png)

그러면 아래와 비슷한 팝업창이 뜬다. 나는 이미 Fork를 했다는 메시지가 뜨지만, Fork 한적 없다면 `[Git Username]/minimal-mistakes`라는 이름의 레파지토리로 Fork 하겠냐는 말이 나왔던것 같다. 그렇다면, 파란 링크를 클릭하자.

![](/assets/images/ETC/Blog/fork_2.png)

마지막으로 Fork한 레파지토리로 들어간다. (아마 자동으로 들어가질거다. 찾아 들어가는 법은.. 굳이 말안해도..ㅋ) 그리고 아까도 설명했지만 반드시 레파지토리의 이름이 `[Git Username].github.io`이어야만 하므로 레파지토리 이름을 수정해준다.

Code, Pull Requests, Actions 등등의 탭이 보일텐데 그 중 `Settings`를 클릭한다. 그럼 바로 Repository name 설정이 보인다! 이 값을 `[Git Username].github.io`로 바꾸고 Rename 버튼을 누르자.

![](/assets/images/ETC/Blog/fork_3.png)

그럼 이제 블로그 생성이 완료 됐다!

잠시 기다리면 Github가 내 블로그를 빌드해서 호스팅 해줄것이다. 몇 분 정도 걸릴수있으며 잠시후 내 블로그 URL로 찾아 들어가면 아래 같은 화면을 볼수있다!

![](/assets/images/ETC/Blog/fork_3.png)

잘 보면 오른쪽 상단에 `Quick-Start Guide`라는 메뉴가 하나 만들어져 있는데 이후의 많은 것들을 그곳의 내용을 보면서 진행하면 된다.

특히, Fork한 레파지토리에서 제거해야할 파일이 몇개 있다. 뭘 제거해야하는지는 [여기](https://mmistakes.github.io/minimal-mistakes/docs/quick-start-guide/#remove-the-unnecessary) 링크를 걸어두겠다. <span class='md-monologue'>(친절친절 ^^)</span>

### 테스트

근데 우리가 블로그 내용을 수정 할 때 마다 수정 내역을 Repository에 업로드 하고 Github가 이를 다시 빌드하고 서비스하는 과정을 반복하도록 할것인가? No~~ 그러면 너무나 비효율적이겠지. 그대신 우리는 블로그를 내 개인 컴퓨터에서 나만 보도록 테스트로 띄워서 수정 내역을 빠르게 확인할 수 있도록 하겠다.

먼저 아마 아래 명령어를 입력해서 mmistakes 테마의 jekyll 블로그를 띄우는데 필요한 패키지를 다운로드 해야 할 것이다.

```shell
bundle install
```

그리고 아래 명령어를 실행해서 테스트 모드로 블로그를 띄워 볼 수 있다.

```shell
bundle exec jekyll serve
```

위 명령이 실행되면 로그를 통해 알 수 있겠지만 기본적으로 웹 브로우저 주소창에 `localhost:4000`을 입력해서 접속 할 수 있다.

### 배포

위의 테스트 작업을 다 거친 후, Git으로 업로드 하자. 역시나  Git Page가 재빌드 되기까지 시간이 걸릴수 있음에 주의하자. 어쩔때는 10분 넘게 걸리기도 하더라.
